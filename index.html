Comparing Windows, macOS, and Linux

KT1901: Files and Directories, Naming Conventions, and File Allocation Table (FAT)

Directory Structure

  • Windows: Uses a hierarchical structure with drives (C:\, D:\) and folders. Paths use backslashes (C:\Users\Name\Documents).
  • macOS: Unix-based structure with a single root directory (/). Paths use forward slashes (/Users/Name/Documents).
  • Linux: Similar to macOS with a root directory (/). Paths use forward slashes (/home/username/Documents).

Naming Conventions

  • Windows: Not case-sensitive by default (FILE.txt = file.txt). Restricted characters: \ / : * ? " < > |
  • macOS: Case-insensitive by default but case-preserving. Fewer naming restrictions than Windows.
  • Linux: Case-sensitive (FILE.txt â‰  file.txt). Restricted characters: / and null character.

File Systems

  • Windows: Primarily uses NTFS, with FAT32 for compatibility (USB drives). FAT is still used for removable media.
  • macOS: Uses APFS (Apple File System) or HFS+. Supports FAT for external drives.
  • Linux: Uses ext4 typically. Supports many file systems including FAT.

KT1902: Executable File Extensions

Executable File Types

  • Windows: .exe (executable), .msi (installer), .bat (batch files), .ps1 (PowerShell scripts)
  • macOS: .app (application bundle), .command (shell scripts), no extension (Unix executables)
  • Linux: No standard extension (permissions determine executability), .sh (shell scripts), .bin (binary)

Script Execution

  • Windows: Runs batch files (.bat) in Command Prompt and PowerShell scripts (.ps1) in PowerShell.
  • macOS: Runs shell scripts in Terminal (bash, zsh). Requires executable permission (chmod +x).
  • Linux: Similar to macOS. Scripts need executable permission. Various shells available (bash most common).
